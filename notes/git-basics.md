Что такое Git?
Git - это распределённая система контроля версий для отслеживания изменений в коде и совместной работы.

Что такое репозиторий?
Репозиторий - это хранилище проекта, где находятся файлы и вся история их изменений.

Зачем нужны коммиты?
Коммиты фиксируют изменения в проекте, позволяя сохранять версии, отслеживать историю и при необходимости откатываться назад.

Что такое Pull Request?
Pull Request - это механизм в GitHub, который позволяет предложить изменения из одной ветки в другую. Обычно используется для проверки кода перед слиянием в основную ветку.

Зачем нужны ветки?
Ветки позволяют:
- разрабатывать новые функции независимо
- исправлять ошибки без влияния на основной код
- работать нескольким разработчикам параллельно
